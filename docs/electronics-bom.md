## BOM по авторской версии модели (TommyB, printables.com)

Автор модели указал следующий список деталей для приводной части
(из описания на https://www.printables.com/model/415497-3d-printed-rc-car):

- 1× вал 3мм×100мм, 2× вал 3мм×30мм
- 12× подшипники 3×7×3мм
- 1× пластиковая шестерня 12T 122A
- 1× сервопривод **MG90s**
- 3× стопорных кольца на вал
- 1× DC-мотор **280**
- Мелкий крепёж (винты M2.5/M3, гайки, шайбы)
- 4× колёсные ступицы под вал 3мм

Автор оригинально проектировал корпус под свои кастомные ESP32-платы, но
**отсек корпуса универсальный ("empty")** — под любую электронику.
Собственный (референсный, не автономный — просто RC по Wi-Fi/ESP-NOW) пример
электроники автора: https://github.com/tommybee456/ESP32-Car-Project
(без лицензии, используется только как справочный материал, не подключён
как submodule и не копируется).

Из его прошивки `esp32Receiver.ino` видно рабочие параметры привода:
- Серво — ШИМ 50 Гц, 10-битный резонанс (`LEDC_TIMER_10_BIT`), пин GPIO9
- DC-мотор — H-мост, ШИМ 1 кГц, пины GPIO5/GPIO7, sleep-пин GPIO4

Эти параметры ШИМ пригодны как отправная точка для настройки нашего
собственного драйвера мотора (см. ниже) — сама прошивка не переиспользуется,
т.к. она реализует ручное RC-управление (ESP-NOW пульт), а не автономное
вождение по камере.

## Предлагаемая электронная архитектура (для автономного вождения)

Требование регламента: **вся обработка — onboard** (см.
`regulation-summary.md`), значит нужен одноплатник, который тянет OpenCV
в реальном времени, а не просто микроконтроллер.

```
                 ┌─────────────────────────────┐
                 │   Raspberry Pi 4B (4/8GB)    │   <- бортовой компьютер,
                 │   или Jetson Nano/Orin Nano  │      камера + вся автономная логика
                 └───────────┬─────────────────┘
                             │ USB / CSI
                     ┌───────┴────────┐
                     │  Камера        │  Raspberry Pi Camera Module 3
                     │  (лента полос) │  (или любая USB-камера с USB-UVC)
                     └────────────────┘
                             │ UART/USB (Serial)
                 ┌───────────┴─────────────────┐
                 │   Микроконтроллер привода    │   Arduino Nano/Uno
                 │   (сервопривод + DC-мотор)   │   или ESP32 (как у автора модели)
                 └─────┬───────────────┬────────┘
                       │ PWM           │ PWM (через H-мост)
               ┌───────┴──────┐  ┌─────┴──────────┐
               │ MG90s Servo  │  │ DC-мотор 280 +  │
               │ (руль,       │  │ H-мост/ESC      │
               │ Print-09)    │  │ (Print-02       │
               │              │  │  Gearbox)       │
               └──────────────┘  └─────────────────┘
```

### Компоненты (стартовый список для закупки)

| Компонент | Назначение | Совместимость с моделью |
|---|---|---|
| Raspberry Pi 4B или Jetson Nano | Бортовая автономная логика (камера → руль/газ) | Помещается в корпус `Print-01` (187×134×28мм) — надо проверить точное крепление под конкретную плату при сборке |
| Raspberry Pi Camera Module 3 (или любая USB-камера) | Детекция полосы/знаков | Крепление — доп. деталь, не входит в оригинальную модель (author использовал свои ESP32-платы без камеры) |
| Arduino Nano / ESP32 | Низкоуровневое управление серво+мотором по командам от Pi | GPIO-параметры ШИМ можно взять по образцу автора (50Гц серво, 1кГц мотор) |
| MG90s Servo Motor | Поворот руля (Ackermann) | Штатная деталь BOM автора, стыкуется с `Print-09_Stearing` |
| DC-мотор 280 + редуктор 12T | Привод задних колёс | Штатная деталь BOM автора, стыкуется с `Print-02_Gearbox` |
| H-мост / простой ESC для щёточного мотора | Управление направлением/скоростью DC-мотора | Не указан автором явно — на выбор (L298N начального уровня либо компактный однока нальный ESC) |
| Li-ion/LiPo аккумулятор (расположение — `Print-13`) | Питание | Подбирается по току мотора 280 и потреблению Pi |

### Что нужно перепроверить руками при сборке (не проверяется по STL автоматически)

- [ ] Точное посадочное место под выбранную модель Raspberry Pi/Jetson внутри
      `Print-01_Main-body` (по STL видно только габарит корпуса, не отверстия
      под конкретную плату — если их нет, потребуется доп. крепёжная пластина)
- [ ] Крепление камеры на корпус — в оригинальной модели автора нет
      кронштейна для камеры (у автора не было автономного вождения по
      камере), нужно спроектировать/напечатать отдельно
- [ ] Соответствие вала мотора 280 диаметру шестерни в `Print-02_Gearbox`
- [ ] Клиренс под аккумулятор в отсеке `Print-13` под конкретный аккумулятор

## Скрипт разбора STL (для повторной проверки размеров)

Пример скрипта, которым посчитаны размеры выше — минимальный парсер
бинарного STL без внешних зависимостей (numpy-stl не был доступен в
окружении на момент анализа):

```python
import struct, glob, os

def read_binary_stl_bbox(path):
    with open(path, 'rb') as f:
        f.read(80)  # header
        n_tri = struct.unpack('<I', f.read(4))[0]
        minx = miny = minz = float('inf')
        maxx = maxy = maxz = float('-inf')
        for _ in range(n_tri):
            data = f.read(50)
            if len(data) < 50:
                break
            vals = struct.unpack('<12fH', data)
            verts = vals[3:12]
            for j in range(0, 9, 3):
                x, y, z = verts[j], verts[j + 1], verts[j + 2]
                minx, maxx = min(minx, x), max(maxx, x)
                miny, maxy = min(miny, y), max(maxy, y)
                minz, maxz = min(minz, z), max(maxz, z)
        return maxx - minx, maxy - miny, maxz - minz

for f in sorted(glob.glob("models/Print-*/*.stl")):
    dx, dy, dz = read_binary_stl_bbox(f)
    print(f"{os.path.basename(f):55s} X={dx:7.1f} Y={dy:7.1f} Z={dz:7.1f}")
```
